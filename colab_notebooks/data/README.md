# Colab Notebooks Data & Evaluation Artifacts

This folder is used to store data files, embeddings, predictions, and metadata generated by the Jupyter Notebooks.

## Recommended File Placements

Move the files downloaded from Google Colab into this directory:

* `cleaned_resumeJD_pairs.csv` — The preprocessed dataset used for training/testing.
* `resume_embeddings.pkl` — The pre-computed resume and JD embeddings.
* `resume_embeddings_with_similarity.csv` — Embedding similarity scores and prediction errors.
* `finetune_test_predictions.csv` — Comparison of base vs. fine-tuned model performance on the test set.
* `finetune_metadata.json` — Hyperparameters, training splits, and metrics (correlation, MAE, RMSE) of the run.

---

> [!NOTE]
> These files are evaluation and training assets. The backend API uses the actual
> SentenceTransformer model folder at
> `backend/ml_models/finetuned_resume_jd_model/`.
>
> To build that folder from `cleaned_resumeJD_pairs.csv`, run:
>
> ```powershell
> venv\Scripts\python.exe scripts\train_resume_jd_model.py
> ```
>
> The fine-tuning notebook creates the same kind of folder with
> `model.fit(..., output_path=OUTPUT_DIR)`, but the original download cell only
> downloaded the CSV/JSON reports. The model folder itself must be zipped and
> downloaded from Colab, or recreated locally with the script above.
